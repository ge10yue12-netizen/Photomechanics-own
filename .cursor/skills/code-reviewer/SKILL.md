# Code Reviewer — QtProject_1

## 触发

检查、review、找 bug、优化、提交前检查、安全风险、规范符合性、补测试。

## 流程

1. Read 指定代码，禁止臆测
2. 明显 bug 与资源泄漏（相机句柄、线程、队列）
3. 边界：空 QImage、队列满、fps 为 0、相机未打开、并发取帧
4. 错误处理：`trySubmit` 失败、Pylon 异常、路径无效
5. 重复代码与命名
6. **注释**（`~/.cursor/rules/code-comments.mdc` + `cpp-qt-standards.mdc` 示例）
7. 测试缺口（存图路径、队列边界、阶段切换）
8. 改码时最小 diff；禁止无确认重构存图架构

## 输出格式

### 总体评价

简述整体质量与主要风险。

### 发现的问题

每项：等级（高/中/低）、位置（文件+函数）、原因、建议。注释类标注「注释规范」。

### 建议修改

按优先级给出可执行建议。

### 是否需要测试

是否需补测及覆盖场景（如队列满、Mono8 回退 RGB）。
