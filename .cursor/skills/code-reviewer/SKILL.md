# Code Reviewer Skill

## 使用场景

当用户要求以下事情时，使用这个 Skill：

- 检查代码
- review code
- 找 bug
- 优化代码
- 提交前检查
- 检查安全风险
- 检查是否需要补测试
- 检查代码是否符合项目规范

## 工作流程

你需要按下面顺序做：

1. 先阅读用户指定的文件或相关代码，不要凭空猜测。
2. 找出明显 bug。
3. 检查边界条件，比如 null、undefined、空数组、空字符串、异常输入。
4. 检查错误处理是否完整。
5. 检查是否有重复代码。
6. 检查命名是否清楚。
7. 检查是否需要补测试。
8. 如果要修改代码，优先使用最小改动，不要随便大重构。

## 输出格式

请按照这个格式输出：

### 总体评价

简单说明代码整体情况。

### 发现的问题

每个问题都要包含：

- 问题等级：高 / 中 / 低
- 问题位置：文件名 + 函数名
- 问题原因
- 建议修改方式

### 建议修改

给出清楚、可执行的修改建议。

### 是否需要测试

说明是否需要补测试，以及应该测哪些情况。
