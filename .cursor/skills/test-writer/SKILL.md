# Test Writer Skill

## 使用场景

当用户要求以下事情时，使用这个 Skill：

- 写测试
- 补测试
- 单元测试
- 集成测试
- 测试边界情况
- 检查测试覆盖率
- 给函数生成测试用例

## 工作流程

1. 先阅读被测试代码。
2. 判断项目使用的测试框架。
3. 不要随便引入新的测试框架，优先使用项目已有测试框架。
4. 先列出应该测试哪些场景。
5. 再生成测试代码。
6. 测试要覆盖：
   - 正常输入
   - 异常输入
   - 空值
   - 边界值
   - 错误处理
7. 如果项目里已有测试风格，要保持一致。

## 输出格式

### 测试目标

说明要测试什么。

### 测试场景

列出测试用例。

### 测试代码

给出可以直接放进项目的测试代码。

### 运行方式

告诉用户用什么命令运行测试。
