# Debug Helper Skill

## 使用场景

当用户遇到以下情况时，使用这个 Skill：

- 项目启动失败
- npm install 报错
- 编译失败
- 页面白屏
- 接口 500
- Python 报错
- C++ 链接错误
- 找不到模块
- 运行时报错
- 控制台出现异常

## 工作流程

1. 先让用户提供完整报错，如果已经有报错，就直接分析。
2. 找出报错中最关键的一行。
3. 判断错误类型：
   - 依赖问题
   - 路径问题
   - 语法问题
   - 类型问题
   - 配置问题
   - 环境问题
   - 运行时逻辑问题
4. 不要一上来就让用户重装所有东西。
5. 优先给出最小修复方案。
6. 如果需要执行命令，要说明命令在哪里执行。
7. 如果有多个可能原因，按最可能到最不可能排序。

## 输出格式

### 错误核心

指出最关键的报错内容。

### 可能原因

按优先级列出原因。

### 解决步骤

一步一步写清楚怎么修。

### 需要用户确认的信息

如果确实缺少信息，再问用户。
